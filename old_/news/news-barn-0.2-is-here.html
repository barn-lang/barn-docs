Barn 0.2v is here!
SolindekDev (Owner) 22-03-2023 20:21
Hi everyone!

New update of Barn is out, it's version 0.2v that have added lot's of cool mechanics and fixed lot's of bugs. Here's full description what did we change in it. This version was made in 2/3 weeks.
Changes:
  • New look of if, elif, else, while, for statements previously it was something like this if |statement| now it's changed to
    if statement. It looks more modern instead of these ugly | characters.
  • Added new auto type which can be used in variable type which means that the compiler decide which type will be the best for this variable. Here's a example of it let random_variable: auto = 1024 // It will be int
  • New types, uhhh that was hard to implemented but now they are working! I'm so happy of this change, I can even say that was the biggest change in 0.2v. Anyway we added brand new types, here are they:
    /* Unsigned types */
    u8  = unsigned char  = uint8_t
    u16 = unsigned short = uint16_t
    u32 = unsigned int   = uint32_t
    u64 = unsigned long  = uint64_t

    /* Normal types */
    i8  = char  = int8_t
    i16 = short = int16_t
    i32 = int   = int32_t
    i64 = long  = int64_t

    /* Float types */
    f32 = float
    f64 = double

    /* Other types */
    string = char*
    bool   = bool
  • Unused warning and new mechanic of used variable. This mechanic works very easy basically if the variable is defined and not used a warning will show up (if you want to disable these warnings use --w-disable-unused) but if the variable name starts with _ for example _example_variable the warning won't show because variables with underscore at the start are the silent variable which don't need to be used. But if you really want to have a variable and don't change it name or deleting it there's a special builtin function named __use__ which will use this variable
  • Added a keyword named extern which can extern functions from C or other compiled languages
  • Now function __code__ can work outside the function body
  • Got back to good old C, so now we are compiling to C not to C++ a little change that in my opinion is the best what i can did. C++ was very hard to compile to it. C is a better option.
  • Format arguments and added fmt_print function to barn which works like printf in C
  • Added new libraries file assert.ba, math.ba, format.ba and limits.ba
  • Constant variables which value can't be modified

These are the most important changes in 0.2v, We are working on 0.3v, in a while you can see here a new post about what we will add to 0.3v. That's all for this post, thanks for reading this whole sh*t.

Have a nice day/night
