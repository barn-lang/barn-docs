Building
Building for Linux will look like this:
git clone https://github.com/barn-lang/barn
cd barn
bash build.sh
barn --help
Or if you want you can download binary releases in our github repository. Getting Barn for Windows will look like this:
git clone https://github.com/barn-lang/barn
cd barn
./build.bat
./barn
If you have any type of errors with building or running barn compiler get help at our discord server.
Hello, World
After getting your barn compiler up and running maybe let start with creating the most easiest program that will just print "Hello, World!". Here's example code of program like that, you can write it by using println function. Here is the code:
@import "std.ba"

fun main() {
 println("Hello, World!")
}
We can see that in the first line we are import std.ba library, Next we are declaring a function named main which is the entry point of every program
Tip!
Without main function your program won't compile, 'cause the error that barn compiler is sending.
and next in the function body we are calling println which will print the "Hello, World" thing and go to the newline. Nothing more complicated! It's that easy..
Comments
In Barn there's two types of comments inline comments and multiline comments. Comments are these parts of code that compiler doesn't see, you can put in them everything and that won't change your program logic. Here is an example of comments:
// This is an inline comment, it will end on newline
/*
 This is an multiline comment, it will end until we will write this
*/
And here is the example how can we use them to document our code (properly):
// Import standard library
@import "std.ba"

// Entry point of this program
fun main() {
  // Ask the user for his name and get it
  println("What's your name")
  let user_name: auto = input()

  // Welcome the user!
  print("Hi ")
  print(user_name)
}
Also a little tip from us :)
Tip!
It is good and smart to comment your code in parts when it's little bit unreadable otherwise your code will be a total mess like Barn Compiler 😊

ARROW // -> After this symbol is type return of function
OPENPARENT // ( Opens arguments
CLOSEPARENT // ) Closes arguments
OPENBRACKET // { Open body function
CLOSEBRACKET // } Close body function
OPENBRACE // [ Open array
CLOSEBRACE // ] Open array

ANDAND // && And
GT // > Great than
GTE // >= Great than of equal
LT // < Less than
LTE // <= Less than of equal
ASN // = Assignment
EQ // == Equals
NEQ // != No equals
PLUS // + Plus
MINUS // - Minus
MUL // * Multiply

Types
In barn we can specify 12 types. Underneath they are all shown with their C/C++ equivalents:
/* Unsigned types */
u8  = unsigned char  = uint8_t
u16 = unsigned short = uint16_t
u32 = unsigned int   = uint32_t
u64 = unsigned long  = uint64_t

/* Normal types */
i8  = char  = int8_t
i16 = short = int16_t
i32 = int   = int32_t
i64 = long  = int64_t

/* Float types */
f32 = float
f64 = double

/* Other types */
string = char*
bool   = bool
Tip!
Difference between unsigned types and normal are that unsigned types can't be smaller than 0. For example u8 is between 0 and 255, but i8 is between -128 and 128.
There's is a file from standard library named limits.ba which defines all the limits of types in barn, here's the example use of it.
@import "limits.ba"
@import "std.ba"

fun main() {
  /*
    Limits.ba source code:
      - https://raw.githubusercontent.com/barn-lang/barn/main/lib/limits.ba
    Here is an example use of u8 max
  */
  let maximum_u8: u8 = _LIMIT_U8_MAX
}
Functions
Functions are the most important thing in basically every programming language. They are just a scrap of code that run when you will call a function like that. Functions can return and get arguments, that are a values with types already specified. Here's an syntax of function declaration in barn:
fun function_name(argument_type argument_name, ...) -> return_type {
  /* Function body */
}
Let's analyse this code, so firstly we use fun keyword which says the compiler that we are trying to declare a function, after it we expect a function_name that will used to give a name for this function (pretty obvious). Next compiler will expect (argument_type argument_name, ...), just a argument list that goes until ) operator. Next function return type which is optional, the look like this arrow (->) and after it correct barn type. And after all good old curly blocks {} in which we write our code! This how example function declaration and function call looks like:
@import "std.ba"

fun foo(i32 foo_argument) -> i32 {
  return foo_argument + 1
}

fun main() {
  let return_of_foo: auto = foo(100)
  printnum(return_of_foo)
}
Tip!
Calling function and program logic can happened only inside a function body
